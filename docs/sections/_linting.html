Linting

The ultimate goal is to have a codebase which looks like the same person wrote it. The reasoning behind this is to promote readable and consistent coding styles. It makes code easier to read, allows all developers to understand the intent behind it, and makes refactoring easier.

scss-lint helps automate this process and outputs any problems with your SCSS, including the offending filename and line number (if available).

The rules of linting are defined in scss-lint.yml file. Should you want to customize the checks run against your code, you can do so by editing this file. You can see the list of all available options in the default configuration file. Refer to to the linters documentation for more information.
